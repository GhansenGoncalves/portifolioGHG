# Testes manuais

Casos de teste, planos de teste e relatórios de bugs de projetos reais
(anonimizados quando necessário).

## Estrutura sugerida

- `test-cases/` — casos de teste por funcionalidade
- `test-plans/` — planos de teste por projeto/release
- `bug-reports/` — relatórios de defeitos com evidências
